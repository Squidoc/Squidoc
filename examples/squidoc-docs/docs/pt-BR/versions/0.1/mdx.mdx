# MDX

Esta página se concentra em arquivos MDX e explica como essa parte funciona em um projeto Squidoc.

## O que você vai configurar

Você vai aprender a descobrir `.mdx` junto de Markdown na árvore de docs.

## O que conferir antes de publicar

Componentes JSX ainda são limitados; confira o build após mudanças em MDX.

## Leia também

[Configuração](/configuration) · [Plugins](/plugins) · [Deploy](/deployment)

## Exemplos

```bash
npx squidoc add plugin @squidoc/plugin-mdx
```

```ts
export default defineConfig({
  plugins: [
    "@squidoc/plugin-seo",
    "@squidoc/plugin-codeblocks",
    "@squidoc/plugin-article-tree",
    "@squidoc/plugin-mdx",
  ],
});
```

```mdx
---
title: API Guide
description: A guide written in an MDX file.
---

# API Guide

Regular Markdown content works here.
```

```mdx
import Callout from "../components/Callout";

<Callout>This is not supported yet.</Callout>
```
