# MDX

Cette page se concentre sur les fichiers MDX et explique comment cette partie fonctionne dans un projet Squidoc.

## Ce que vous allez configurer

Vous allez apprendre à découvrir `.mdx` à côté de Markdown dans les docs.

## À vérifier avant publication

Les composants JSX restent limités; vérifiez le build après chaque changement MDX.

## À lire aussi

[Configuration](/configuration) · [Plugins](/plugins) · [Déploiement](/deployment)

## Exemples

```bash
npx squidoc add plugin @squidoc/plugin-mdx
```

```ts
export default defineConfig({
  plugins: [
    "@squidoc/plugin-seo",
    "@squidoc/plugin-codeblocks",
    "@squidoc/plugin-article-tree",
    "@squidoc/plugin-mdx",
  ],
});
```

```mdx
---
title: API Guide
description: A guide written in an MDX file.
---

# API Guide

Regular Markdown content works here.
```

```mdx
import Callout from "../components/Callout";

<Callout>This is not supported yet.</Callout>
```
